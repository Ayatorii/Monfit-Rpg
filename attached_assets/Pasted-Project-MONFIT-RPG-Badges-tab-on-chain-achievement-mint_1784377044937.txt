## Badge definitions
3 badges, images attached (GREAT_WARRIOR.png, TASK_MASTER.png, WALLET_CONNECTOR.png). Display names come directly from the image content:
- **Great Warrior** — requirement: 100 total Arena battles fought (win or lose, just total matches played) reaches 100.
- **Task Master** — requirement: 100 total daily quests completed reaches 100.
- **Wallet Connector** — requirement: wallet is connected and signed in (this one is trivially eligible immediately after a successful SIWE sign-in — no counter needed).

2. Total Arena battles = count of rows in the existing `matches` table for that wallet — no new column needed, just a count query.
3. New contract `AchievementBadge.sol` (ERC-721, OpenZeppelin, same Foundry project already set up for SeasonTrophy):
   - Owner-gated `mintBadge(address player, uint8 badgeType) external onlyOwner returns (uint256 tokenId)` where badgeType is an enum/uint8 (0 = WalletConnector, 1 = TaskMaster, 2 = GreatWarrior).
   - Prevent double-minting: track `mapping(address => mapping(uint8 => bool)) hasMinted` (or similar) and revert if the player already has that badge type.
   - Store badgeType per tokenId so it can be read back later.

5. New backend routes:
