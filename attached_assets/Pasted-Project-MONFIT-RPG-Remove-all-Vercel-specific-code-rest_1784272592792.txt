## 3. Keep all of these working fixes/changes (do not lose them in the revert)
- **Loading Screen**: simple splash with logo/title/tagline + a "Start" button only — no wallet UI here. Clicking Start goes straight into the app (Train tab) in guest/local mode.
- **Wallet Connect in the header**: a persistent header element (visible across all 5 tabs) showing "Connect Wallet" when disconnected, or the shortened wallet address when connected, with disconnect option. This triggers the RainbowKit modal + SIWE flow.
- **Clean RainbowKit + wagmi setup**: `getDefaultConfig` from RainbowKit, `monadTestnet` from `viem/chains` (chainId 10143), WalletConnect project ID from env (`VITE_WALLETCONNECT_PROJECT_ID`).
- **Sign-message call uses wagmi's `signMessageAsync` hook** (connector-aware) — confirmed NOT a direct `window.ethereum.request(...)` call. Keep it this way; this is correct and should not regress.
- **Hard timeout on the sign request** (~30s) with a clear "Signature request failed or timed out — try again" fallback UI instead of hanging indefinitely, plus a way to cancel/continue as guest from the stuck state.
- **Guard against auto-reconnect silently re-triggering signing**: only start the SIWE flow on explicit user-initiated connect, not on wagmi's silent reconnect on page load.
- **Nonce route explicitly calls `req.session.save()` before responding** (prevents the nonce/session race condition where verify arrives before the nonce is durably written).
- **Sessions table created via a proper Drizzle migration file**, not manual SQL (this needs to actually be reproducible — confirm this is true now, since it's been a recurring issue).
- **Cross-origin-safe session cookie config**: `sameSite: "none"`, `secure: true` on the session cookie, and `credentials: "include"` on every frontend fetch call to the API — this was needed because Replit's dev preview serves frontend and backend on different origins (iframe-based). Confirm this is intact and wasn't reset by the architecture migration — check `custom-fetch.ts` (Orval-generated client) specifically, since codegen can silently drop manual patches.
- **Database connection**: whatever connection method (pg Pool or otherwise) is currently used is fine for Replit's persistent-server model — no need for `@neondatabase/serverless`'s HTTP driver anymore since we're not in a serverless environment. Just confirm the connection is stable and doesn't hang or leak connections over time in a long-running process.
