Project: MONFIT RPG — Restructure wallet connect: move out of Loading Screen, into the app itself

Two changes combined:

## 1. Loading Screen becomes simple — no wallet logic here at all
- Remove ALL wallet connect UI, RainbowKit modal triggers, sign-in flow, and auto-reconnect logic from the Loading Screen entirely.
- Loading Screen now shows: logo, title, tagline, and a single "Start" button.
- Clicking "Start" navigates directly into the app (Train tab) in guest/local mode — no wallet involved at this point at all. This removes the fragile "wallet auto-reconnect fires immediately on page load" race condition entirely, since nothing wallet-related runs until the user is already deep in the app and explicitly asks for it.

## 2. Wallet Connect moves into the app's persistent header
- Add a "Connect Wallet" element to the persistent header/top bar that's visible across all 5 tabs (Train/Character/Shop/Arena/Leaderboard) — same header area that already shows Gold/XP.
- When not connected: show a compact "Connect Wallet" button in the header.
- When connected: show a shortened wallet address (e.g. `0x1234...abcd`) with a way to disconnect (dropdown or click-to-disconnect).
- Clicking "Connect Wallet" here opens the RainbowKit modal and runs the exact same SIWE flow as before (nonce → sign → verify → session), just triggered from this new location instead of the Loading Screen.
- On successful connect: fetch the wallet's saved Gold/XP/Level/inventory from the database and merge it into the current session (replacing local guest state) — same behavior as before, just triggered later in the user's session rather than at app launch.
- If the user was playing as guest and connects a wallet mid-session with existing local guest progress, keep the existing migration behavior (migrate local guest progress into the new database row) if that was already implemented — confirm and preserve this.

## 3. Clean reinstall of RainbowKit + wagmi (same as requested before, do this fresh regardless)
