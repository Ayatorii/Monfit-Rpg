Lessons from the Step 1 audit — apply these from the start this time, don't wait for a fix pass:
- Any progress/percentage bars or animated fills: animate `transform: scaleX()`, never animate `width` (layout thrash).
- All interactive/dynamic elements need proper ARIA (nav needs `role="navigation"` + `aria-label`, active tab needs `aria-current="page"`, any progress indicators need `role="progressbar"` with `aria-valuenow`/min/max).
- Respect `prefers-reduced-motion` for any entrance/transition animations (reuse the MotionConfig pattern already set up in Step 1).
- No hard-coded hex colors outside the token system — every color used must trace back to a CSS variable.
- No Tailwind default palette colors (slate/gray/indigo defaults) anywhere — everything must come from the MONFIT RPG token set.
- No decorative blur() glows over ~60px — expensive on mobile GPU; use flat radial-gradient backgrounds instead if atmosphere is needed.

Navigation shell
- Mobile (`< md`): fixed bottom nav bar, 5 icon+label tabs, current tab visually distinct (violet accent), safe-area-inset padding for iOS home indicator.
- Desktop (`≥ md`): left sidebar, same 5 items, can show full labels (not just icons).
- Use React Router (or the routing already present in the project) — each tab is a real route (`/train`, `/character`, `/shop`, `/arena`, `/leaderboard`), not just a state toggle, so routing/deep-linking works correctly.
- Icons: pick a consistent icon set (lucide-react is fine) — do not mix icon styles across tabs.
