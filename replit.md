# MONFIT RPG

A fitness-themed RPG web app with wallet-based progress saving on Monad Testnet. Players connect a wallet (MetaMask, Rabby, or any WalletConnect-compatible mobile wallet), sign in via SIWE, then train, shop, fight in the Arena, and climb the Leaderboard — all progress persisted in PostgreSQL keyed by wallet address.

## Run & Operate

- The app runs via two Replit-managed workflows: `artifacts/monfit-rpg: web` (frontend, React + Vite) and `artifacts/api-server: API Server` (Express backend). Use the `WorkflowsRestart` tool to (re)start them — do not run `pnpm dev` manually at the repo root.
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only; production schema is managed by Replit Publish)

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React 19 + Vite 7, Tailwind CSS, shadcn/ui, Framer Motion
- Wallet: wagmi v2 + RainbowKit v2 (Monad Testnet, chainId 10143)
- Auth: SIWE (Sign-In With Ethereum) via viem/siwe
- API: Express 5
- DB: PostgreSQL + Drizzle ORM — tables: `users`, `players`, `matches`, `player_items`, `session`
- Sessions: express-session + connect-pg-simple (cookie: `SameSite=None; Secure`)
- API codegen: Orval (from OpenAPI spec at `lib/api-spec/openapi.yaml`)
- Build: esbuild (CJS bundle)

## Architecture decisions

- **SIWE over JWT**: sessions stored server-side in PostgreSQL via connect-pg-simple; avoids token leakage and supports server-side revocation.
- **SameSite=None + Secure on all cookies**: the Replit preview pane is an iframe on a different origin (`replit.com`), making all API calls cross-site. Without this the nonce cookie never reaches /verify.
- **`monadTestnet` from `viem/chains`**: never hand-written — chainId 10143 is taken directly from the library to avoid misconfiguration.
- **Optimistic client state + server sync**: `game-context.tsx` updates local state immediately and fires API calls in the background; guests play with pure client state.
- **OpenAPI-first codegen**: all API types and React Query hooks are generated by Orval from `lib/api-spec/openapi.yaml`; never edit `lib/api-client-react/src/generated/` by hand.

## Post-merge setup

`scripts/post-merge.sh` runs automatically after a task agent merge:
1. `pnpm install --frozen-lockfile` — installs dependencies
2. `pnpm --filter @workspace/db run push` — applies any schema changes to the dev database

## Gotchas

- The `session` table must exist before the API server can store nonces. If the API server crashes with a pg relation error, run `pnpm --filter @workspace/db run push`.
- `createTableIfMissing: false` in `session.ts` — connect-pg-simple does NOT auto-create the table; Drizzle owns it.
- All frontend fetch calls go through `lib/api-client-react/src/custom-fetch.ts` which always sets `credentials: "include"` — do not bypass this with raw `fetch()`.
- The nonce is single-use. After /verify the session is regenerated (new session ID) to guard against session fixation.
